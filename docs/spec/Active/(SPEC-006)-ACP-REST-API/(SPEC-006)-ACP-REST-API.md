# ACP REST API

## Summary

Implement a localhost HTTP server that exposes ACP endpoints for external callers (scripts, CI pipelines, other agents). This is the public interface of the harness.

## Acceptance Criteria

### AC-1: POST /acp/call
**Given** a valid ACP request body
**When** POST /acp/call is received
**Then** the request is validated, routed through the command router, and the ACP response is returned

### AC-2: GET /acp/status
**Given** the harness is running
**When** GET /acp/status is received
**Then** it returns pool status, active requests, and recent call history

### AC-3: POST /acp/verify
**Given** a valid ACP request with type `verify`
**When** POST /acp/verify is received
**Then** the request is routed through the verification loop
**And** the teardown report is returned

### AC-4: GET /acp/reports
**Given** verification reports exist
**When** GET /acp/reports is received
**Then** it returns a list of teardown report summaries

### AC-5: Error Responses
**Given** an invalid request or internal failure
**When** the error occurs
**Then** the API returns structured JSON errors with appropriate HTTP status codes

## Implementation Approach

Use FastAPI with uvicorn for the HTTP server. Bind to `localhost:8420` by default (configurable). Middleware validates ACP schema (SPEC-002), routes to CommandRouter (SPEC-003), and handles errors. Async throughout.

## Scope

- `src/rest_api.py` — FastAPI application, routes, middleware
- OpenAPI schema auto-generated by FastAPI
